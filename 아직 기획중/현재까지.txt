메이플 데일리 로그 현재 구상 메모
작성일: 2026-07-08
갱신일: 2026-07-09

1. 전체 구조

- 서버: Nexon API 호출, 데이터 저장, 리포트 계산 담당
- 앱: 화면/UI, 게임 실행 감지, 플레이 시간 기록, 수동 갱신 요청 담당
- DB: SQLite 로컬 파일 사용
- 1차 MVP는 추천/일일 리포트에 필요한 데이터 수집과 저장부터 구현

2. 현재 확정된 방향

- 서버 기술은 Django REST Framework를 사용한다.
- 앱 기술은 Flutter를 사용한다.
- DB는 Supabase/PostgreSQL을 사용하지 않고 SQLite를 사용한다.
- Supabase URL, anon key, service role, RLS 정책은 현재 구조에서 필요 없다.
- 서버는 `server/.env`를 자동으로 읽는다.
- 서버는 필요한 SQLite 테이블을 API 호출 시 자동 생성한다.
- 앱은 DB에 직접 붙지 않고 DRF 서버의 조회 API를 호출한다.
- 강제 새로고침은 앱이 서버에 요청하고, 서버가 Nexon API를 호출한 뒤 SQLite에 저장한다.

3. 서버 역할

- Nexon OpenAPI 호출
- 계정 캐릭터 목록 동기화
- 캐릭터별 스냅샷 저장
- 최신 스냅샷 조회 API 제공
- 전날 마지막 스냅샷과 오늘 마지막 스냅샷 비교 기반의 report 생성
- Nexon API 호출 제한 관리

4. 앱 역할

- 실제 화면/UI 제공
- 로컬 PC에서 게임 실행/종료 감지
- 게임을 켠 날 기준으로 플레이 시간 저장
- 플레이 시간은 분 단위로 저장
- 강제 새로고침 버튼 제공
- 서버에서 캐릭터 목록과 최신 스냅샷을 조회

5. 보안

- Nexon API key는 앱에 넣지 않는다.
- ADMIN_TOKEN은 서버 API 보호용으로 사용한다.
- 개인 앱이므로 1차에서는 복잡한 사용자 인증은 하지 않는다.
- `.env`, SQLite DB 파일, 로그 파일은 Git에 올리지 않는다.

6. Nexon API 호출 제한

- 초당 최대 5건
- 하루 최대 1,000건
- 1차에서는 강제 새로고침과 캐릭터 동기화 중심으로 호출한다.
- 필요하면 나중에 예약 동기화나 호출량 관리 로직을 추가한다.

7. 리포트 기준

- 리포트 테이블은 `reports` 단일 테이블로 사용한다.
- `report_type`으로 `daily`, `weekly`, `monthly`를 구분한다.
- 1차는 `daily`만 구현한다.
- 일일 리포트는 전날 마지막 스냅샷과 해당 날짜 마지막 스냅샷을 비교한다.
- 리포트에는 시작 스냅샷 ID와 종료 스냅샷 ID를 저장한다.
- 원본 API 응답은 `character_snapshots.snapshot_json`에 저장한다.

8. 1차 테이블 후보

- `characters`: 캐릭터 기본 정보, ocid, 월드, 직업, 태그, 기록 제외 여부
- `character_snapshots`: API 호출 시점의 캐릭터 상태 원본 JSON
- `starforce_events`: 스타포스 로그
- `cube_events`: 큐브 로그
- `potential_events`: 잠재 재설정 로그
- `play_time_records`: 날짜별 총 플레이 시간
- `reports`: daily/weekly/monthly 리포트 결과

9. 현재 구현된 API

- `GET /health`
- `GET /api/characters`
- `GET /api/characters/<character_id>/latest-snapshot`
- `POST /api/sync/characters`
- `POST /api/sync/snapshot`
- `POST /api/reports/daily`

10. 관련 파일

- `server/`: Django REST Framework 서버
- `server/.env.example`: 필요한 환경변수 예시
- `database/schema.sql`: SQLite 기준 참고 스키마
- `docs/nexon-api-rules.md`: Nexon API 호출 제한 정리
- `docs/report-rules.md`: 리포트 계산 규칙

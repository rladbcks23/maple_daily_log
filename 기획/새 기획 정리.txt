메이플 숙제알리미 새 기획 정리
작성일: 2026-07-13

프로젝트명: 메이플 숙제알리미

1. 전체 구조

- 서버와 앱의 분리 형태는 유지한다.
- 기본 성격은 일반적인 알람 앱에 가깝게 잡는다.
- 기존 메이플 도움 페이지처럼 정보를 조회하는 서비스가 아니라, 사용자가 놓친 이벤트나 숙제를 알려주는 서비스로 기획한다.
- 알림은 데스크톱 기본 알림 형태가 아니라, 백그라운드에서 실행 중인 앱이 화면에 켜지는 방식으로 제공한다.
- 최우선 기능은 숙제/이벤트/공지 알림이다.
- 리포트처럼 데이터를 비교하거나 계산해서 보여주는 기능은 1차 범위에서 제외한다.
- 추가 기능은 알림 기능을 먼저 완성한 뒤 필요하면 후순위로 검토한다.

2. 놓친 이벤트 예시

일반적으로 메이플 스케줄러에 등록되는 정보들을 우선 대상으로 잡는다.

2-1. 일간 이벤트

- 일일퀘스트
- 일일보스
- 몬스터 파크
- 오늘 로그인 정보

2-2. 주간 이벤트

- 주간퀘스트
- 주간보스
- 익스트림 몬스터 파크
- 에픽 던전
- 심볼퀘
- 길드
- 수로
- 플래그

2-3. 월간 이벤트

- 검은 마법사 보스

2-4. 일정 기간마다 존재하는 이벤트

- 진행 중인 출석 이벤트
- 진행 중인 주간 이벤트
- 선데이 메이플

2-5. 업데이트될 때 안내할 정보

- 공지사항
- 캐시샵 공지사항
- 업데이트

3. 서버 역할

서버는 사용자가 직접 확인하지 않아도 되는 정보를 정해진 시간에 수집하고, 앱이 알림 판단에 사용할 수 있는 형태로 정리한다.

3-1. 매일 수행할 작업

- 매일 밤 9시에 오늘 로그인 정보를 확인한다.
- 매일 밤 9시에 일일 이벤트 정보를 확인한다.
- 일일 이벤트에는 일일퀘스트, 일일보스 등을 포함한다.

3-2. 매주 수행할 작업

- 매주 화요일 또는 수요일 밤 9시에 주간 이벤트 정보를 확인한다.
- 주간 이벤트에는 주간퀘스트, 주간보스, 주간 단위 이벤트 등을 포함한다.

4. 앱 역할

앱은 서버에서 수집한 정보를 바탕으로 사용자에게 필요한 알림을 제공한다.

- 오늘 접속하지 않았으면 접속 알림을 준다.
- 오늘 해야 할 일일퀘스트, 일일보스, 이벤트 등이 남아 있으면 알림을 준다.
- 이번 주에 하지 않은 보스, 퀘스트, 이벤트가 있으면 알림을 준다.
- 메이플 런처의 종료를 감지한다.
- 런처 종료 시 당일 일일 이벤트 완료 여부를 확인한다.
- 등록된 일일 이벤트를 모두 완료했다면 알림을 주지 않는다.
- 하나라도 완료하지 않은 이벤트가 있다면 앱을 화면에 띄워 알림을 제공한다.

5. 다음 정리 예정

- 필요한 Nexon API 목록은 별도 문서로 정리한다.
- 각 이벤트별로 API에서 직접 확인 가능한 정보와 수동 등록이 필요한 정보를 구분한다.
- 앱이 알림을 띄우는 기준과 예외 상황을 세부적으로 정리한다.
- 리포트, 성장 비교, 장비 비교 같은 계산형 기능은 알림 기능 완성 전까지 진행하지 않는다.

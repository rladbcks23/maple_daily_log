메이플 숙제알리미 기획 정리
작성일: 2026-07-13

프로젝트명: 메이플 숙제알리미

1. 전체 구조

- 서버와 앱의 분리 형태는 유지한다.
- 넥슨 로그인을 활용한다.
- 일반적인 알람 앱에 가까운 구조로 만든다.
- 정보 조회 서비스가 아니라, 사용자가 놓친 이벤트와 숙제를 알려주는 앱으로 만든다.
- 알림은 데스크톱 기본 알림이 아니라, 백그라운드에서 실행 중인 앱이 화면에 켜지는 방식으로 제공한다.
- 최우선 기능은 숙제, 이벤트, 공지 알림이다.
- 리포트처럼 데이터를 비교하거나 계산해서 보여주는 기능은 만들지 않는다.
- 선택한 캐릭터 정보는 DB에 저장한다.
- 서버는 선택한 캐릭터 정보를 기준으로 API를 호출하고 앱에 전달한다.
- 알림 판단 기준은 `docs/알림 판단 규칙.md`에 정리한다.
- 추가 기능은 알림 기능을 먼저 만든 뒤 필요하면 다시 검토한다.

2. 놓친 이벤트 예시

스케줄러에 등록된 정보들을 우선 대상으로 잡는다.

2-1. 일간

- 일일퀘스트
- 일일보스
- 몬스터 파크
- 오늘 로그인 정보

2-2. 주간

- 주간퀘스트
- 주간보스
- 익스트림 몬스터 파크
- 에픽 던전
- 심볼퀘
- 길드
- 수로
- 플래그

2-3. 월간

- 검은 마법사 보스

2-4. 일정 기간마다 존재하는 정보

- 진행 중인 출석 이벤트
- 진행 중인 주간 이벤트
- 선데이 메이플

2-5. 업데이트될 때 안내할 정보

- 공지사항
- 캐시샵 공지사항
- 업데이트

3. 서버 역할

서버는 정해진 시간에 필요한 정보를 수집하고, 앱이 알림을 판단할 수 있는 형태로 정리한다.
1차에서는 스케줄러/공지 API 응답 원본을 장기 저장하지 않고, 알림 판단에 필요한 응답을 앱에 전달하는 역할에 집중한다.
- 넥슨 계정의 캐릭터 목록을 조회한다.
- 선택한 캐릭터의 `ocid`를 조회한다.
- 선택한 캐릭터의 기본 정보를 조회한다.
- 런처 종료 감지 후 앱 요청이 들어오면 선택한 캐릭터의 스케줄러 정보를 조회한다.

3-1. 매일

- 매일 밤 9시에 오늘 로그인 정보를 확인한다.
- 매일 밤 9시에 일일 이벤트 정보를 확인한다.
- 단, 오늘 해당 캐릭터로 접속하지 않았다면 스케줄러 API의 일일 콘텐츠 정보가 비어 있을 수 있다.
- 접속 전에는 일일 이벤트 완료 여부를 확정하지 않고, 접속 필요 알림을 우선한다.
- 일일 이벤트에는 일일퀘스트, 일일보스 등을 포함한다.

3-2. 매주

- 매주 화요일 또는 수요일 밤 9시에 주간 이벤트 정보를 확인한다.
- 주간 정보는 화요일과 수요일에만 조회한다.
- 주간 콘텐츠 정보도 접속 이후에 채워질 가능성이 있으므로, 빈 배열을 완료 상태로 판단하지 않는다.
- 주간 이벤트에는 주간퀘스트, 주간보스, 주간 단위 이벤트 등을 포함한다.

4. 앱 역할

- 서버에서 수집한 정보를 가지고 알림 역할을 한다.
- 넥슨 계정에 있는 캐릭터 중 알림에 사용할 일부 캐릭터를 선택할 수 있게 한다.
- 선택한 캐릭터 정보는 `selectedCharacters`로 저장한다.
- 저장 정보는 캐릭터명, 월드명, `ocid`, 직업, 레벨, 캐릭터 이미지, 선택 시각이다.
- 메이플 런처 종료 시 선택 캐릭터 정보를 갱신한다.
- 캐릭터 선택 화면에서는 사진, 월드, 캐릭터명, 직업, 레벨을 보여준다.
- 오늘 접속하지 않았으면 알림을 준다.
- 접속 전 스케줄러 정보가 비어 있으면 미완료 목록 대신 접속 필요 알림을 준다.
- 오늘 해야 할 일일퀘스트, 일일보스, 이벤트 등이 남아 있는 것이 확인되면 알림을 준다.
- 이번 주에 하지 않은 보스, 퀘스트, 이벤트가 있으면 알림을 준다.
- 메이플 런처의 종료를 감지한다.
- 런처 종료 시 당일 일일 이벤트 완료 여부를 확인한다.
- 등록된 일일 이벤트를 모두 완료했다면 알림을 주지 않는다.
- 하나라도 완료하지 않은 이벤트가 있다면 앱을 화면에 띄워 알림을 제공한다.

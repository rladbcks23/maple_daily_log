# 알림 판단 규칙

메이플 숙제알리미의 알림 조건을 정리한다.
알림은 사용자가 놓칠 수 있는 일간/주간 숙제, 보스, 공지, 이벤트를 알려주는 데 집중한다.

## 1. 매일 밤 9시 알림

### 조건

- 선택한 캐릭터의 오늘 스케줄러 정보를 조회한다.
- 오늘 접속하지 않은 것으로 보이거나 `daily_contents`가 비어 있으면 접속 필요 상태로 본다.

### 알림

- 오늘 접속하지 않았다면 접속해서 오늘 숙제를 하라고 알림을 준다.
- 알림 대상은 일간 콘텐츠와 일일 보스를 우선한다.

### 주의

- `daily_contents`가 비어 있다고 해서 모든 일간 숙제를 완료한 것으로 판단하지 않는다.
- 미접속 상태에서는 일간 콘텐츠 목록이 비어 있을 수 있으므로, 미완료 목록 대신 접속 필요 알림을 우선한다.

## 2. 매주 화/수 밤 9시 알림

### 조건

- 선택한 캐릭터의 이번 주 스케줄러 정보를 조회한다.
- 주간 콘텐츠와 보스 콘텐츠의 완료 여부를 확인한다.

### 알림

- 이번 주 주간 콘텐츠나 보스를 완료하지 않았다면 목요일 전까지 하라고 알림을 준다.
- 화요일과 수요일 밤 9시를 주간 숙제 확인 시점으로 사용한다.

### 주의

- `weekly_contents`도 접속 이후에 채워질 가능성이 있다.
- 주간 콘텐츠가 빈 배열로 내려오면 완료 상태로 확정하지 않는다.

## 3. 주기적 정보 업데이트 알림

### 조건

- 공지사항, 진행 중 이벤트, 캐시샵 공지, 업데이트 목록을 주기적으로 확인한다.
- `lastSeenNotices`와 비교해 새 글이나 변경된 정보가 있는지 확인한다.

### 알림

- 새 공지나 이벤트가 있으면 알림을 준다.
- 진행 중 이벤트 기간, 캐시샵 판매 기간, 업데이트 등록일을 기준으로 알림 여부를 판단한다.
- 이미 `lastSeenNotices`에 저장된 공지는 다시 알림으로 띄우지 않는다.
- 새 공지를 알림으로 보여준 뒤 `lastSeenNotices`를 갱신한다.

## 4. 메이플 런처 종료 시 알림

### 조건

- 메이플 런처 종료를 감지한다.
- 런처 종료 후 선택한 캐릭터의 스케줄러 정보를 다시 조회한다.
- 일간 콘텐츠 또는 일일 보스 중 완료하지 않은 항목이 있는지 확인한다.

### 알림

- 오늘 일간 콘텐츠나 일일 보스를 하지 않은 것이 확인되면 알림을 준다.
- 등록된 일간 콘텐츠와 일일 보스를 모두 완료했다면 알림을 주지 않는다.

### 확인 불가

- 런처 종료 후에도 `daily_contents`가 비어 있으면 완료 여부를 확정하지 않는다.
- 이 경우에는 "스케줄러 정보를 확인할 수 없음" 상태로 처리한다.

## 5. 공통 판단 기준

- 빈 배열은 완료 상태가 아니다.
- 접속 전 빈 배열은 확인 불가 또는 접속 필요 상태로 본다.
- 알림은 확인 가능한 미완료 항목이 있을 때만 구체적인 목록을 보여준다.
- 완료 여부를 확정할 수 없으면 완료로 처리하지 않는다.

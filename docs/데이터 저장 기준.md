# 데이터 저장 기준

메이플 숙제알리미의 1차 저장 대상은 사용자가 알림 대상으로 선택한 캐릭터다.
리포트, 성장 기록, 장비 비교용 데이터는 저장하지 않는다.

## 1. 1차 방향

- 선택한 캐릭터 정보는 DB에 저장한다.
- 서버는 Nexon OpenAPI를 호출하고 결과를 앱에 전달한다.
- 스케줄러/공지 API 응답은 알림 판단에 필요한 범위에서만 사용한다.
- 리포트, 성장 기록, 장비 비교용 데이터는 저장하지 않는다.

## 2. selectedCharacters

사용자가 알림 대상으로 선택한 캐릭터 목록을 저장한다.

저장 필드:

- `characterName`
- `worldName`
- `ocid`
- `characterClass`
- `characterLevel`
- `characterImage`
- `selectedAt`

## 3. 갱신 기준

- 앱 종료 시 선택 캐릭터 정보를 갱신한다.
- 메이플 런처 종료 감지 후 알림 판단을 하기 전에 선택 캐릭터 정보를 기준으로 API를 호출한다.
- 캐릭터 기본 정보가 바뀔 수 있으므로 종료 시점마다 최신 기본 정보를 반영한다.

## 4. OCID 저장 기준

- `ocid`는 캐릭터 스케줄러 API 호출에 필요하므로 저장한다.
- `ocid`는 인증 토큰이나 API key가 아니므로 단독으로 API 호출 권한을 주지는 않는다.
- 다만 캐릭터 식별자이므로 외부 로그, 공개 저장소, 화면 공유에 불필요하게 노출하지 않는다.

## 5. 저장하지 않는 데이터

- 일일/주간 리포트
- 캐릭터 성장 기록
- 장비 비교 기록
- 스냅샷 원본 응답
- 강화/큐브/잠재 이력

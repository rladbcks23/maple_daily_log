# 데이터 저장 기준

메이플 숙제알리미의 1차 저장 대상은 사용자가 알림 대상으로 선택한 캐릭터와 마지막으로 확인한 공지 정보다.
리포트, 성장 기록, 장비 비교용 데이터는 저장하지 않는다.

## 1. 1차 방향

- 선택한 캐릭터 정보는 DB에 저장한다.
- 마지막으로 확인한 공지 정보는 DB에 저장한다.
- 서버는 Nexon OpenAPI를 호출하고 결과를 앱에 전달한다.
- 스케줄러/공지 API 응답은 알림 판단에 필요한 범위에서만 사용한다.
- 리포트, 성장 기록, 장비 비교용 데이터는 저장하지 않는다.

## 2. selectedCharacters

사용자가 알림 대상으로 선택한 캐릭터 목록을 저장한다.

저장 필드:

- `characterName`
- `worldName`
- `ocid`
- `characterClass`
- `characterLevel`
- `characterImage`
- `selectedAt`

## 3. 갱신 기준

- 메이플 런처 종료 시 선택 캐릭터 정보를 갱신한다.
- 메이플 런처 종료 감지 후 알림 판단을 하기 전에 선택 캐릭터 정보를 기준으로 API를 호출한다.
- 캐릭터 기본 정보가 바뀔 수 있으므로 메이플 런처 종료 시점마다 최신 기본 정보를 반영한다.
- 스케줄러 API에서 일일/주간 콘텐츠가 빈 배열로 내려와도 완료 상태로 저장하거나 판단하지 않는다.
- 접속 전 빈 배열은 확인 불가 상태로 보고, 필요하면 접속 필요 알림으로 처리한다.

## 4. lastSeenNotices

공지, 이벤트, 캐시샵 공지, 업데이트의 중복 알림을 막기 위해 마지막으로 확인한 공지 정보를 저장한다.

저장 필드:

- `noticeType`
- `noticeId`
- `title`
- `registeredAt`
- `lastSeenAt`

공지 종류:

- `notice`
- `event`
- `cashshop`
- `update`

사용 기준:

- 목록 조회 결과의 식별자를 기준으로 새 글 여부를 판단한다.
- 이미 저장된 `noticeId`는 다시 알림으로 띄우지 않는다.
- 새 글을 알림으로 보여준 뒤 `lastSeenNotices`를 갱신한다.
- 상세 조회가 필요한 경우에만 본문, 이벤트 기간, 판매 기간을 추가로 확인한다.

## 5. OCID 저장 기준

- `ocid`는 캐릭터 스케줄러 API 호출에 필요하므로 저장한다.
- `ocid`는 인증 토큰이나 API key가 아니므로 단독으로 API 호출 권한을 주지는 않는다.
- 다만 캐릭터 식별자이므로 외부 로그, 공개 저장소, 화면 공유에 불필요하게 노출하지 않는다.

## 6. 저장하지 않는 데이터

- 일일/주간 리포트
- 캐릭터 성장 기록
- 장비 비교 기록
- 스냅샷 원본 응답
- 강화/큐브/잠재 이력

# 데이터 저장 기준

메이플 숙제알리미의 저장 기준은 공통 데이터와 개인 로컬 데이터를 분리한다.
모든 유저에게 동일한 데이터는 서버 DB에 저장하고, PC마다 달라지는 데이터는 앱 로컬 캐시에 저장한다.

## 1. 서버 DB 저장

서버 DB는 전체 유저가 공유해도 되는 데이터만 저장한다.

- 공지/이벤트/캐시샵/업데이트 스냅샷
- 이번주 썬데이 메이플 정보

서버 DB에 저장하는 이유:

- 공지와 이벤트는 모든 유저에게 공통이다.
- Nexon API 호출 횟수를 줄일 수 있다.
- 새 공지/이벤트 판단 기준을 서버에서 일관되게 관리할 수 있다.
- 앱을 재설치하거나 다른 PC에서 실행해도 공통 데이터 기준은 유지된다.

## 2. noticeSnapshots

새 공지 알림을 위해 가장 최근에 수집한 공지 목록을 저장한다.
이 데이터는 읽음 표시가 아니라, 이전 수집 목록과 현재 수집 목록을 비교하기 위한 기준이다.

저장 필드:

- `noticeType`
- `noticeId`
- `title`
- `link`
- `registeredAt`
- `collectedAt`

공지 종류:

- `notice`
- `event`
- `cashshop`
- `update`

사용 기준:

- 주기적으로 공지 목록 API를 호출한다.
- 가장 최근에 저장한 `noticeSnapshots`와 지금 받아온 공지 목록을 비교한다.
- 이전 수집 목록에는 없고 현재 목록에 새로 생긴 공지만 알림으로 띄운다.
- 이전 수집 목록에 있었지만 현재 목록에서 없어진 공지는 신경 쓰지 않는다.
- 비교가 끝나면 현재 공지 목록으로 `noticeSnapshots`를 갱신한다.
- 공지 읽음 표시는 제공하지 않는다.

## 3. sundayEventSnapshot

이번주 썬데이 메이플 정보를 저장한다.
진행 중 이벤트 목록에서 썬데이 메이플이 사라져도 마지막으로 확인한 썬데이 정보를 유지하기 위한 서버 기준 데이터다.

저장 필드:

- `noticeId`
- `title`
- `link`
- `registeredAt`
- `thumbnail`
- `eventStartAt`
- `eventEndAt`
- `content`
- `contentImageUrls`
- `collectedAt`

사용 기준:

- 진행 중 이벤트에 썬데이 메이플이 있으면 상세 정보를 저장한다.
- 진행 중 이벤트에 썬데이 메이플이 없으면 서버 DB에 저장된 최신 썬데이 정보를 반환한다.
- 강제 새로고침 시 종료된 이벤트 목록에서 최신 썬데이 메이플을 다시 조회해 갱신한다.
- 앱 로컬 썬데이 캐시는 서버 응답 실패 시 표시용 백업으로만 사용한다.

## 4. 앱 로컬 캐시 저장

PC마다 달라지는 데이터는 앱 로컬 캐시에 저장한다.

- 선택된 캐릭터 목록
- 캐릭터 목록
- 캐릭터 이미지
- 캐릭터별 스케줄러
- 대시보드 통계
- 알림 설정
- 알림 발송 기록

앱 로컬 캐시에 저장하는 이유:

- 넥슨 로그인이나 사용자 식별자가 없으므로 서버에서 개인 선택 상태를 구분할 수 없다.
- 선택 캐릭터와 스케줄러는 PC/사용자마다 달라진다.
- API 응답이 비어 있거나 실패했을 때 마지막 정상 데이터를 임시 표시할 수 있다.
- 캐릭터 생성, 레벨 변경, 이미지 변경은 API 조회 시 로컬 캐시를 갱신한다.

## 5. 저장하지 않는 데이터

- 서버 DB의 선택 캐릭터 목록
- 공지 읽음 표시
- 캐릭터 성장 기록
- 장비 비교 기록
- 강화/큐브/잠재 이력
- 일일/주간 리포트

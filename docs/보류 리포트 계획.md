# Report And Scheduler Plan

이 문서는 이전 리포트/스냅샷 중심 기획을 보관한 보류 문서다.
현재 최우선 기능은 메이플 숙제알리미의 알림 기능이며, 리포트처럼 데이터를 비교하거나 계산해서 보여주는 기능은 당장 구현하지 않는다.
리포트 기능은 알림 기능을 먼저 완성한 뒤 추가 기능으로 다시 검토한다.

이 문서는 앱이 백그라운드에서 실행 중이라는 전제를 기준으로 리포트 생성, 하루 정리, 스케줄러 알림 규칙을 정리한다.

## 전제

- 앱은 최소 백그라운드에서 실행 중이라고 가정한다.
- 게임 실행/종료를 감지할 수 있어야 한다.
- 게임 시작 시점과 종료 시점을 기준으로 플레이 타임을 기록한다.
- Nexon API 스냅샷은 리포트와 하루 정리에 필요한 비교 기준으로 사용한다.
- 리포트 작성 시간은 우선 새벽 5시를 기준 후보로 둔다.
- 실제 메이플스토리 초기화 시간과 앱 정산 시간은 설정값으로 분리한다.

## 리포트 종류

### 일일 리포트

- 생성 주기: 매일
- 대상 기간: 하루 전날
- 생성 시간 후보: 새벽 5시
- 비교 기준:
  - 전날의 마지막 스냅샷
  - 전전날의 마지막 스냅샷
- 기록할 내용:
  - 경험치 증가량
  - 총 플레이 타임
    - 시작 시간
    - 총 플레이타임
  - 전투력 변화
  - 전날과 비교해 달라진 모든 변경점
  - 장비 변경점
  - 스탯 변경점
  - 심볼 변경점
  - V매트릭스 변경점
  - HEXA 변경점
  - 하이퍼스탯 변경점
  - 어빌리티 변경점
  - 링크 스킬 변경점
  - 펫/안드로이드/외형 변경점
  - 스케줄러 완료 상태 요약

### 주간 리포트

- 생성 주기: 매주 목요일
- 생성 시간 후보: 새벽 5시
- 대상 기간: 최근 1주일
- 비교 기준:
  - 리포트 생성 대상 주의 마지막 스냅샷
  - 1주일 전 기준 스냅샷
- 기록할 내용:
  - 일주일 전과 비교해 달라진 모든 변경점
  - 경험치 증가량
  - 총 플레이 타임
  - 전투력 변화
  - 장비 변경점
  - 스탯/심볼/V매트릭스/HEXA 변경점
  - 강화/큐브/잠재 이력 요약
  - 주간 퀘스트/주간 보스 완료 상태 요약

### 월간 리포트

- 생성 주기: 매월 1일
- 생성 시간 후보: 새벽 5시
- 대상 기간: 최근 1개월
- 비교 기준:
  - 리포트 생성 대상 월의 마지막 스냅샷
  - 1개월 전 기준 스냅샷
- 기록할 내용:
  - 1달 전과 비교해 달라진 모든 변경점
  - 경험치 증가량
  - 총 플레이 타임
  - 전투력 변화
  - 장비 변경점
  - 스탯/심볼/V매트릭스/HEXA 변경점
  - 강화/큐브/잠재 이력 요약
  - 월간 성장 요약

## 하루 정리

### 생성 시점

- 게임 종료 감지 시 생성한다.
- 게임 종료 직후 최신 스냅샷을 저장한 뒤, 게임 시작 전 또는 직전 스냅샷과 비교한다.

### 보여줄 내용

- 해당 세션의 플레이 타임
- 경험치 증가량
- 전투력 증가량
- 주요 변경점
- 스케줄러에 등록된 미완료 일일 퀘스트
- 조건부 미완료 주간 퀘스트

### 주간 퀘스트 조건

- 화요일 또는 수요일에 게임을 종료한 경우:
  - 스케줄러에 등록된 주간 퀘스트 중 미완료 항목을 같이 보여준다.
- 수요일 밤 9시:
  - 스케줄러에 등록된 주간 퀘스트 중 미완료 항목이 있으면 알림을 보낸다.

## 백그라운드 스케줄러 역할

앱은 리포트 생성뿐 아니라 간단한 스케줄러 역할도 수행한다.

### 필요 기능

- 게임 실행 감지
- 게임 종료 감지
- 플레이 타임 기록
- 게임 시작 전 스냅샷 저장
- 게임 종료 후 스냅샷 저장
- 매일 새벽 정산 작업
- 매주 목요일 주간 리포트 생성
- 매월 1일 월간 리포트 생성
- 수요일 밤 9시 주간 퀘스트 미완료 알림
- API 호출 제한을 고려한 스냅샷 범위 조절

## 스냅샷 정책

### 수집 트리거

데이터 수집은 아래 이벤트를 기준으로 실행한다.

- 앱 실행 시:
  - 수집 타입: `app_start`
  - 기본 번들: `light`
  - 목적: 앱이 켜졌을 때 캐릭터 목록과 최소 상태를 빠르게 확인한다.
- 메이플 런처 실행 시:
  - 수집 타입: `maple_launcher_start`
  - 기본 번들: `light`
  - 목적: 플레이 시작 전 기준 스냅샷을 만든다.
- 메이플 런처 종료 시:
  - 수집 타입: `maple_launcher_end`
  - 기본 번들: `full`
  - 목적: 플레이 종료 후 하루 정리와 변경점 비교에 사용할 전체 스냅샷을 만든다.
- 갱신 이벤트 버튼 클릭 시:
  - 수집 타입: `force_refresh`
  - 기본 번들: `full`
  - 목적: 사용자가 명시적으로 요청한 최신 상태를 저장한다.
- 1시간마다 자동 수집:
  - 수집 타입: `hourly_auto`
  - 기본 번들: `daily`
  - 목적: 앱이 켜져 있는 동안 중간 상태를 주기적으로 저장한다.

### 수집 번들

- `light`:
  - 캐릭터 기본 정보
  - 능력치
  - 스케줄러 상태
- `daily`:
  - 캐릭터 기본 정보
  - 능력치
  - 장착 장비
  - 장착 심볼
  - HEXA 정보
  - 스케줄러 상태
- `full`:
  - 캐릭터 기본 정보
  - 능력치
  - 하이퍼스탯
  - 어빌리티
  - 장착 장비
  - 캐시 장비
  - 장착 심볼
  - 적용 세트
  - 스킬 정보
  - V매트릭스
  - HEXA 정보
  - HEXA 스탯
  - 기타 능력치 영향 요소
  - 장착 중인 시드링
  - 예비 특수 반지
  - 스케줄러 상태
- `history`:
  - 스타포스 이력
  - 잠재능력 재설정 이력
  - 큐브 이력

### 캐릭터 분류

- `본캐`:
  - 일반 수집 대상
  - 일일 리포트 수집 대상
  - 주간 리포트 수집 대상
- `부캐`:
  - 일일 리포트 작성에 필요할 때만 수집한다.
- `주보돌이`:
  - 주간 리포트 작성에 필요할 때만 수집한다.
- `ignore`:
  - 자동/일괄 수집에서 제외한다.

새로 동기화된 캐릭터의 기본 태그는 `ignore`로 둔다.
사용자가 명시적으로 분류한 캐릭터만 자동/일괄 수집 대상에 포함한다.
캐릭터 하나에는 태그를 최대 하나만 적용한다.

### 게임 시작 시

- 가벼운 스냅샷을 저장한다.
- 추천 범위:
  - 캐릭터 기본 정보
  - 능력치
  - 스케줄러 상태

### 게임 종료 시

- 하루 정리에 필요한 스냅샷을 저장한다.
- 추천 범위:
  - 캐릭터 기본 정보
  - 능력치
  - 장비
  - 심볼
  - HEXA
  - 스케줄러 상태

### 정기 리포트 생성 전

- 리포트 정확도를 위해 필요한 경우 최신 스냅샷을 저장한다.
- API 호출 제한에 걸릴 수 있으므로 모든 캐릭터가 아니라 주요 캐릭터 중심으로 실행한다.

## 구현 메모

- 일일/주간/월간 리포트는 같은 계산 구조를 공유하고 비교 기간만 다르게 둔다.
- 하루 정리는 리포트보다 즉시성이 중요하므로 게임 종료 이벤트에 묶는다.
- 새벽 5시는 리포트 생성 시간으로 쓰고, 실제 게임 초기화 기준 시간은 별도 설정으로 둔다.
- 알림은 사용자가 끄거나 시간을 바꿀 수 있게 만드는 것이 좋다.
- 미완료 퀘스트 알림은 스케줄러 API에 등록된 항목만 기준으로 한다.
